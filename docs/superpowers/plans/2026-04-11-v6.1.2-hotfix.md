## Task 6: Make `noise_filter()` scan-type aware

**Files:**
- Modify: `canadian_llm_council_brain.py:2983` (method signature)
- Modify: `canadian_llm_council_brain.py:2989` (call to get_historical_edge_multiplier)
- Modify: `canadian_llm_council_brain.py:4863` (call site)

**Why this matters:** Without this fix, a ticker with <53% accuracy on MORNING picks gets dropped from EVENING scans too. The noise filter uses aggregate accuracy across all scan types. A ticker could be noisy for intraday prediction but strong for overnight prediction (or vice versa), and the current code can't distinguish.

- [ ] **Step 1: Add optional scan_type parameter to noise_filter()**

Change line 2983 from:
```python
    def noise_filter(self, tickers: list[str]) -> list[str]:
```
to:
```python
    def noise_filter(self, tickers: list[str], scan_type: str | None = None) -> list[str]:
```

- [ ] **Step 2: Pass scan_type to get_historical_edge_multiplier()**

Change line 2989 from:
```python
            multiplier = self.get_historical_edge_multiplier(ticker)
```
to:
```python
            multiplier = self.get_historical_edge_multiplier(ticker, scan_type=scan_type)
```

`get_historical_edge_multiplier()` already accepts `scan_type` (added in v6.1.2 Task 8) and passes it to `get_ticker_accuracy()` which already has the filtered query. No further changes needed in that chain.

- [ ] **Step 3: Pass scan_type at call site in run_council()**

The call site at line 4863 is in Step 4b of `run_council()`. The `trigger` parameter is available from the method signature but `scan_type` is computed later at line 5596. We need to compute it earlier or use `trigger` directly.

The simplest approach: compute scan_type from trigger at the call site.

Change lines 4863-4864 from:
```python
            clean_tickers = self.historical_analyzer.noise_filter(
                [p.ticker for p in payloads_list]
            )
```
to:
```python
            _scan_type = trigger.upper() if trigger.upper() in ("MORNING", "EVENING") else "MORNING"
            clean_tickers = self.historical_analyzer.noise_filter(
                [p.ticker for p in payloads_list], scan_type=_scan_type
            )
```

- [ ] **Step 4: Commit**

```bash
git add canadian_llm_council_brain.py
git commit -m "fix: noise_filter scoped by scan_type

A ticker noisy in MORNING picks (< 53% accuracy) was incorrectly
dropped from EVENING scans too. Now filters using scan-type-specific
accuracy, allowing tickers to perform differently across scan types.

Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>"
git push origin main
```
